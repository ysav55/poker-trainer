  • Pure function: no state, no side effects
  • 7-card → best 5-card hand detection

  • Wheel straight (A-2-3-4-5) edge case
Called by: GameManager._resolveShowdown()
🟢 SidePotCalculator
  • Pure function: no state, no side effects
  • Builds side pot breakdown from contribution levels

Called by: GameManager._resolveShowdown()
🟢 HandGenerator
  • Fill-the-Gaps algorithm
  • null = fill randomly, string = use this card
  • Validates card formats and duplicates
  • Builds collision-free shuffled deck for remaining slots

🟣 Deck.js
  • createDeck() — 52 deterministic cards
  • shuffleDeck() — Fisher-Yates
  • isValidCard() — format validator
  • getUsedCards() — duplicate prevention

4 Raise Logic — The Most Complex Path

Raise validation has 5 layers of guards and two distinct outcomes depending on whether the raise is "full" or an incomplete all-in.

✅ Valid Raise Allowed When:
  • It's this player's turn (current_turn matches)
  • Game is not paused
  • Amount ≥ current_bet + min_raise, OR player is going all-in for less
  • Player has enough chips for the total commitment
  • If previous raise was incomplete: only the original raiser can re-raise
🚫 Raise Blocked When:
  • Not this player's turn
  • Game is paused
  • Amount is below minimum and player still has chips left
  • Previous raise was an incomplete all-in AND this player already acted this street AND is not the original raiser
⚡ Full Raise (≥ min_raise)
  • min_raise updated to the new raise increment
  • All other active players' action reset to "waiting" — must act again
  • last_raise_was_full = true
  • last_aggressor = this player
🔵 Incomplete All-In (< min_raise)
  • min_raise stays the same (can't shrink it)
  • Other players' action NOT reset (round still closes normally)
  • last_raise_was_full = false
  • Subsequent players may call/fold only (no re-raise except original raiser)
5 Identified Gaps & Problems
HIGH Blind levels changeable mid-hand
setBlindLevels() has no phase guard — the coach can change small/big blinds while a hand is running. Blinds already posted won't be adjusted, but min_raise (reset each street to big_blind) would silently change, creating inconsistent raise minimums within the same hand.
⚠ Impact: corrupts raise validation mid-hand
HIGH Dealer rotation skips no one
On resetForNextHand(), dealer seat increments by 1 regardless of who's connected or in-hand. If a player with in_hand=false or disconnected=true is next in the rotation, they still receive the dealer button and have blinds posted against them — even if they're meant to sit out.
⚠ Impact: disconnected/sitting-out players become dealer
MED Manual card injection works in any mode
manualDealCard() has no guard against being called in rng mode. A coach could manually override a card that was dealt from the shuffled deck, creating a state where the deck array and the actual dealt cards are inconsistent. The card is replaced in the player slot but the deck still "contains" that card in its remainder.
⚠ Impact: deck state inconsistency in rng mode
MED No stack persistence on reconnect
When a player disconnects and the 60s TTL expires, they're removed via removePlayer(). If they reconnect after the TTL, addPlayer() gives them stack=1000 (default). Their actual stack from the previous session is lost. There's no mechanism to restore a player's previous stack when they re-join within the same table session.
⚠ Impact: chip counts reset for late re-joiners
MED SessionManager doesn't track 3bet/aggression levels
trackPreflopAction() uses boolean flags — a player who opens raises and then 3-bets (calls a re-raise) would have vpipThisHand=true, pfrThisHand=true, same as someone who just opened. The system can't distinguish who was the initial raiser vs who 3-bet vs who called. Aggression frequency and 3-bet % are untracked.
⚠ Impact: missing key poker training metrics
MED Snapshot deep-clone on every action
Every call to _saveSnapshot() deep-clones the entire game state object — all players, board, deck remainder, config. With 9 players and a 30-action history, you could hold 30 full-state copies in memory simultaneously. No structural sharing or lazy clone — it's a full JSON.parse(JSON.stringify(state)) equivalent on every action.
⚠ Impact: memory pressure on long hands; CPU spikes at 30-action depth
MED awardPot() has no winner validation
The coach can manually award the pot to any player ID — including a folded player, a player who sat out, or a non-existent ID. There's no guard that checks whether the target is an eligible winner. The system will silently award the pot and mark them as winner even if they folded preflop.
⚠ Impact: pot can be awarded to ineligible/invalid players
RESOLVED notifications[] dead state
Field verified absent in current codebase — ISS-33 closed 2026-03-16 as non-issue.
✓ Closed
LOW No starting stack configuration
addPlayer() always initializes with stack=1000. There's no way for the coach to configure starting stacks at table creation time. The workaround is adjustStack() per player after joining — an extra manual step per player.
Low — UX friction only

Real-Time Equity Engine
Run a Monte Carlo simulation server-side on every street transition. Store the equity percentages (each player's probability of winning) in the game state. Coach sees live equity during pauses — turns abstract coaching ("you should have folded") into hard numbers ("you had 18% equity here").
  • Calculate on each street advance: preflop, flop, turn, river
  • Use 5,000-sample Monte Carlo — fast enough to run synchronously
  • Expose equity per player in game_state (coach-only field)
  • Store final equity snapshot in hands table for post-session review
  • Power a new "Expected Value" column in the Stats Dashboard

3
Decision Quality Scorer
After each player action, score the decision quality independently of the result — a bad call can win, a good fold loses. Score based on pot odds vs equity, position, stack-to-pot ratio, and board texture. Aggregate into a "decision quality" metric that separates skill from luck over time.
  • Runs after equity is computed (depends on Idea 1)
  • Pot-odds check: is the call price ≤ equity × pot?
  • Position check: is the player in or out of position?
  • New column in hand_actions: decision_score (A/B/C/D or 0–100)
  • New Stats Dashboard widget: average decision score per player per session
  • Separates "bad beat recipient" from "bad decision maker"
Depends on Idea 1 New: DecisionScorer.js Medium effort
⏱️
4
Decision Timing Profiler
Track how long each player takes per decision (time from turn start to action). Store it alongside each action in the DB. Build a per-player timing histogram. Players who act instantly on tough spots (tell: no thinking) or take forever on easy spots (tell: tanking for deception) get flagged. Coaches can use this as a training tool for timing tells.
  • Server stores turn_started_at when current_turn is set
  • New column in hand_actions: think_time_ms — gap between turn start and action
  • Stats Dashboard: "Avg think time by street" and "Think time vs decision type" charts
  • Flag outliers: <2s on a river all-in decision, >50s on a preflop fold
  • Zero changes to GameManager — computed and stored in the index.js action handler
