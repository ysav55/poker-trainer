Poker Trainer

Real-time poker training tool. A coach controls the game; players join from any browser and act in turn. All hand history and stats persist to Supabase (PostgreSQL).

Node.js 18+ · Express · Socket.io React + Vite + Tailwind Supabase (PostgreSQL) 910 tests passing Last updated 2026-03-17
1

What it does

Six core capabilities that make it a training tool, not just a poker game.

🎓
Coach-led sessions
  • Coach controls deal, pace, undo, pause
  • Configure specific hole cards or ranges per player
  • Force-next-street, rollback, award pot manually
  • Adjust blind levels and stacks between hands
🃏
Live hand play
  • 60-second action timer with auto-fold
  • Full betting: fold, check, call, raise, all-in
  • Side-pot support for multi-player all-ins
  • BB-view toggle persisted to localStorage
🎬
Guided replay
  • Step through any historical hand action-by-action
  • All hole cards exposed for coaching discussion
  • Branch to live play from any point in the replay
  • Unbranch back to the saved replay state
📊
Stats & analytics
  • VPIP / PFR / WTSD / WSD / 3-bet% per player
  • Career stats hover cards on every seat
  • Session HTML report (auto-generated)
  • 30+ auto-tags: 3BET_POT, C_BET, WHALE_POT…
🗂️
Hand config & ranges
  • Pin specific cards or constrain by range tags
  • Board texture constraints (monotone, paired, etc.)
  • Combo intersection with live count badge
  • Playlists for curated drill sequences
🔐
Closed-system auth
  • Roster managed in players.csv
  • Coach role granted by CSV entry, not env var
  • Spectator mode: name-only, no DB record
  • Stable UUID per player persisted in Supabase

Accounts & Access

Closed system — no self-registration. Admin edits players.csv at the project root.

players.csv format

# Poker Trainer — Player Roster # Format: name,password,role # role must be: coach or student Coach,coach123,coach Alice,alice123,student Bob,bob123,student

Rules

  • Add a row → grant access immediately on next restart
  • Delete a row → revoke access (restart or call reload())
  • Role must be exactly coach or student
  • Names are case-insensitive for login; capitalisation in file is shown in-game
  • Server exits fatally at startup if players.csv is missing

Logging in

🔑 Player / Coach

  • Open the app → Log In tab
  • Enter name + password from roster
  • Server returns a stable UUID stored in localStorage
  • Coach flag is granted automatically by CSV role

👁 Spectator

  • Click Watch tab
  • Enter any display name — no password needed
  • No DB record created; cannot act or bet
  • Sees the table live but opponents' cards stay hidden
⚠ ISS-76: loginRosterPlayer uses exact case match. Ensure names in players.csv match exactly what players type at login.

Coach Controls

11 sections in the right-side sidebar. Only visible when logged in with a coach role.

§1 — Game Controls
  • Start Hand — deal (random or configured)
  • Configure Hand — open card config phase
  • Pause / Resume — freeze action timer
  • Force Next Street — skip betting round
  • Undo — roll back last player action
  • Rollback Street — revert to street start
§4 — Blind Levels
  • Change SB / BB between hands only
  • Default: SB=5, BB=10 → 1,000 starting stack
  • Any player joining after a blind change gets 100×BB
  • BB must be greater than SB
§5 — Player Management
  • ✕/✓ toggle to exclude a player from next hand
  • Excluded flag resets automatically after hand starts
  • OFFLINE badge shown for disconnected players
  • Adjust Stack: directly set any player's chips
§6 — Session Report
  • Opens a full HTML report in a new browser tab
  • VPIP/PFR/WTSD/WSD comparison across all players
  • Pattern summary and mistake flags
  • Key hand breakdowns for the session
§8 — Live Hand Tags
  • Click tag buttons during a hand to annotate it
  • Tags: 3BET_POT, C-BET, CHECK_RAISE, BLUFF_CATCH…
  • Saved to Supabase; visible in Stats Dashboard
  • Manually tagging auto-creates a playlist for that tag
§9 — Playlist Manager
  • Create named playlists of hands for drill sessions
  • Activate a playlist → Reset Hand loads next scenario
  • Auto-playlist from tags: tag a hand → playlist created
  • Same hand can appear in multiple playlists
§10 — Scenario Loader
  • Search hands by player, date, or tags
  • Load — pre-fills config for a fresh replay
  • Replay — enters Guided Replay Mode
  • Add results to a playlist for batch drill
§11 — Replay Controls
  • Scrubber + Back/Forward step buttons
  • Branch to Live: play from current replay state
  • Return to Replay: undo live branch
  • Exit Replay: restores pre-replay stacks

§6 — Configuring a Hand

Hand Configuration Flow

  • Click Configure Hand → config phase opens (only between hands)
  • Per player: toggle Cards (pin specific hole cards) or Range (constrain by tags)
  • Range groups — radio within group, intersection across groups: PAIRS (All/QQ+/77-JJ/22-66) · SUIT (Suited/Offsuit) · TYPE (Broadway/Connectors/1-Gap/Ace-high/King-high) · SHORTCUT (ATo+/KJo+/Premium/Strong)
  • A combo count badge shows live intersection size; 0 combos = falls through to random
  • Board texture constraints: Suit (Rainbow/Flush Draw/Monotone) · Pair (Unpaired/Paired/Trips) · Connect (Connected/Disconnected) · High (Broadway/Low/Ace High)
  • Server retries flop generation up to 100 times to satisfy all constraints

Playing

What players see and can do during a hand.

🃏 During a hand

  • Your hole cards are visible only to you (HIDDEN to others)
  • Action panel slides up when it's your turn
  • Actions: Fold · Check · Call · Raise (slider + ½pot / 1×pot / 2×pot / all-in)
  • 60-second timer — auto-folds on expiry
  • Disconnecting during your turn pauses the timer for 60 s

📊 Stats hover cards

  • Hover any seat to see VPIP / PFR / WTSD / WSD / 3-bet% / Alltime Winning
  • Shows both session stats and all-time career stats
  • Alltime Winning displays 0 if the total is negative
  • Fetched from Supabase on first hover; cached for the page session

🔢 BB view toggle

  • Chips / BB pill button in top-left corner of the table
  • Switches between flat amounts (e.g. "1,000") and BB units (e.g. "100bb")
  • Personal setting — only affects your view
  • Remembered across sessions via localStorage

🔌 Disconnection

  • Amber OFFLINE badge at 50% opacity on disconnected seat
  • If it's their turn: timer pauses; 60 s to reconnect
  • After 60 s with no reconnect: player removed, game continues
  • Coach disconnect: game pauses for all players until coach returns

Guided Replay Mode

phase = 'replay' built into GameManager — not a separate class. Accessed via §10 Scenario Loader in the coach sidebar.

ControlWhat it does
Progress label"Action N / Total" — describes player, street, action type, amount at current step
ScrubberDrag to jump directly to any action in the hand
◀ Back / Fwd ▶Step backward or forward one action at a time
Branch to LiveSnapshot state at cursor → switch to live play. BRANCHED badge (amber) replaces REPLAY badge (blue).
Return to ReplayUnwind the live branch; restore the exact step where you branched
Exit ReplayEnd replay; restore pre-replay stacks; return to WAITING lobby

Key behaviours

  • All hole cards are exposed to coach during non-branched replay (coachInReview = true)
  • Betting controls hidden during non-branched replay; reappear when branched to live
  • Purple ring on player seat whose turn it is at the current replay step
  • Loading a replay while a playlist is active is blocked (prevents state corruption)
  • pre_branch_snapshot is stripped from undo history entries — prevents exponential memory growth

Raise Logic

✅ Valid Raise — Allowed When

  • It's this player's turn (current_turn matches)
  • Game is not paused
  • Amount ≥ current_bet + min_raise, OR player going all-in for less
  • Player has enough chips for total commitment
  • If previous raise was incomplete: only original raiser can re-raise

🚫 Raise Blocked — When

  • Not this player's turn
  • Game is paused
  • Amount below minimum and player still has chips
  • Previous raise was incomplete all-in AND this player already acted AND is not original raiser

⚡ Full Raise (≥ min_raise)

  • min_raise updated to new raise increment
  • All other active players' action reset to "waiting" — must act again
  • last_raise_was_full = true

🔵 Incomplete All-In (< min_raise)

  • min_raise stays the same
  • Other players' action NOT reset — round closes normally
  • Subsequent players may call/fold only (no re-raise)

Data Layer — Supabase

All persistence is Supabase (PostgreSQL). SQLite has been fully removed. The server uses the service role key; the browser client uses the anon key.

  • is_roster INTEGER (1 = active in CSV)

  • All hands link back here

  • hand_id (UUID PK) · session_id FK
  • board · final_pot · winner_id/name
  • auto_tags · mistake_tags · coach_tags
  • small_blind · big_blind · dealer_seat
👥
hand_players
  • PK (hand_id, player_id)
  • stack_start / stack_end · hole_cards
  • is_winner · vpip · pfr · wtsd · wsd
  • seat · position (BTN/SB/BB/UTG…)
⚡
hand_actions
  • id (auto) · hand_id FK · player_id
  • street · action · amount · timestamp
  • is_reverted · is_manual_scenario
  • stack_at_action · pot_at_action

playlists + playlist_hands
  • playlists: name · table_id · created_at
  • playlist_hands: playlist_id + hand_id (PK)

  • CASCADE DELETE on playlist removal

  • timestamp of manual stack change
  • Used for Alltime Winning calculation

Key design decisions

Stable player identity

  • player_id in ALL tables is the client's stable UUID — NOT socket.id
  • UUID stored in localStorage under key poker_trainer_player_id
  • Server stableIdMap: Map<socketId, stableId> populated on join_room
  • Coach stableId is deterministic coach_<tableId> (not a UUID — skips upsertPlayerIdentity)
  • join_room validates registered stableId for non-coach/non-spectator players

RLS & access policy

⚠ ISS-75: Supabase anon read policies (USING (true)) on all tables are temporary (Phase 3 measure). Any unauthenticated client can read all hand/player data. Acceptable for local/private use. Replace with user-scoped RLS policies in Phase 1 (Auth Migration) before internet-facing deployment.

Issues Registry

🔴 CRITICAL — crash or wrong outcome  ·  🟠 HIGH — broken feature  ·  🟡 MEDIUM — inconsistency / tech debt  ·  🟢 LOW — minor / cosmetic

ISS-74
HIGH
Coach sidebar overlays table — rightmost seat partially obscured

Sidebar is position: fixed, 18 rem wide. Full fix requires flex layout to push main content. Partial mitigation: seat 7 moved from 95%→88%. Workaround: collapse sidebar during play.

ISS-73
MEDIUM
Replay hole cards missing for hands recorded before ISS-69 fix (2026-03-16)

Pre-fix hands stored socket.id as player_id; replay lookup keys on stableId so they never match. New hands are unaffected. Mitigation: use Load Scenario instead of Replay for old hands.

ISS-75
MEDIUM
Supabase anon read policies are open (USING (true))

Temporary Phase 3 measure — any unauthenticated client can read all hand/player data. Acceptable for local/private training use. Replace with user-scoped RLS in Phase 1 (Auth Migration) before internet-facing deployment.

ISS-76
MEDIUM
loginRosterPlayer is case-sensitive on display_name

Uses .eq('display_name', name) — exact match. If players.csv capitalisation differs from what the player types, login returns "not found". Full fix: case-insensitive collation or lowercase normalisation on upsert + lookup.
