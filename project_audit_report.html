Security & Risks

CRITICAL
✅

RESOLVED: Passwords now hashed with bcrypt · Secret fallback removed

All passwords in players.csv were migrated to bcrypt hashes (cost factor 12) via scripts/hash-passwords.js. PlayerRoster.authenticate() now uses bcrypt.compare() — plain-text passwords are never stored or compared.

Also resolved: The hardcoded 'dev-secret-change-in-production' fallback was removed. The server now refuses to start (process.exit(1)) if SESSION_SECRET is not set, preventing silent use of a known-weak secret in production.

  • • Game actions validated before state changes
  • • Coach actions require verified coach role, not client-supplied flag

  • • ❌ Plain-text passwords in players.csv

  • • ❌ Hardcoded fallback secret in server code
  • • Fix: ~2–4 hours to hash passwords with existing bcrypt library

CRITICAL — RESOLVED Plain-text passwords replaced with bcrypt hashes

CRITICAL — RESOLVED Hardcoded SESSION_SECRET fallback removed

The 'dev-secret-change-in-production' fallback was removed from server/index.js. The server now calls process.exit(1) on startup if SESSION_SECRET is missing, preventing silent use of a known-weak secret. A CORS_ORIGIN warning was also added for production deployments without an explicit origin.

✓
HIGH — RESOLVED .env.example created with documentation for all required vars

A new .env.example file documents every environment variable with descriptions, generation instructions for secrets, and security notes. New developers can now configure the project correctly without guessing what's needed.

✓
HIGH — RESOLVED Health check now verifies database connectivity

The GET /health endpoint was upgraded from a static {status:"ok"} to an async handler that pings Supabase (player_profiles table). It now returns HTTP 503 + {status:"degraded"} if the DB is unreachable, making it suitable as a real liveness probe for deployment platforms.

✓
MEDIUM — RESOLVED (ISS-76) Case-sensitive login fixed

The loginRosterPlayer DB lookup in HandLoggerSupabase.js was changed from .eq('display_name', name) to .ilike('display_name', name), making the Supabase lookup case-insensitive. A player typing "coach" or "COACH" now correctly finds the "Coach" record in the database.

✓
MEDIUM — RESOLVED (ISS-74) Coach sidebar overlay fixed — now pushes the table layout

The CoachSidebar component's root div was changed from position: fixed to a normal flex item (h-full flex flex-shrink-0). The sidebar now occupies its own column in the flex row, so the poker table naturally shrinks when the panel is open instead of being covered by it.

✓
LOW — RESOLVED docker-compose.yml added + Dockerfile cleaned up

A docker-compose.yml was added with two profiles: prod (single container build) and dev (nodemon server with volume mounts). The Dockerfile was also cleaned up — dead SQLite environment variables and outdated comments from the pre-Supabase era were removed.

✓
FEATURE — ADDED Auto-sleep / auto-wake — server shuts down when idle, wakes on demand

Two complementary mechanisms prevent runaway hosting costs on Fly.io free tier. In-app: server/index.js reads IDLE_TIMEOUT_MINUTES; after that many minutes with zero Socket.io connections it gracefully marks all hands incomplete and calls process.exit(0). Platform: fly.toml already had auto_stop_machines = "stop" / auto_start_machines = true; the new IDLE_TIMEOUT_MINUTES = "20" env var wires the in-app timer to the same policy. Fly wakes the VM in ~1 s on the next incoming request.
