01 — Coach-Driven

Manual Tagging

The coach can label any active hand with a preset tag at any point while the hand is running. These labels survive forever and are never overwritten by the system.

1
Hand is in progress
A "TAG HAND" pill appears at the top-right of the screen. It's only visible during an active hand — it disappears in lobby or replay mode.
2
Coach clicks the pill
The pill expands to show six preset tag chips: Review, Bluff, Hero Call, Mistake, Key Hand, 3-Bet Pot. Clicking a chip toggles it on or off.
3
Tags are saved
Each selection immediately sends a update_hand_tags event to the server. The server deletes any existing coach tags for that hand and inserts the new set. A "✓ Saved" indicator confirms success.
4
A playlist is auto-created
For each tag applied, the system creates or reuses a playlist with that tag's name, then adds this hand to it. Tag "Bluff" → a "Bluff" playlist is ready for drill sessions immediately. No manual playlist work needed.
5
Pill collapses on hand end
When the coach resets the hand, the pill auto-collapses and clears its selection for the next hand. Coach tags on the previous hand remain stored forever.
i
Coach tags are the only tags that are never replaced by the system. They persist indefinitely regardless of how many times the hand is re-analyzed.

What makes it coach-only
The server checks the player's role before processing update_hand_tags. Anyone without a coach role gets an error. Regular players cannot tag hands.
Replace, not append
Every save replaces all existing coach tags for that hand. Removing a chip and saving removes it from the database. There is no separate "delete" action.
02 — System-Driven

Auto-Tagging

When a hand ends, one pipeline runs automatically — no coach involvement needed. It builds a rich context object, then fires nine specialized analyzers in parallel, each responsible for a different dimension of the hand.

⚠
There is one auto-tagging pipeline, not two separate systems. However, it produces three distinct tag types — auto, mistake, and sizing — which behave and display differently. This may be the source of the "two taggers" impression.
Trigger
Coach resets hand
The moment the coach clicks "Reset Hand", the hand is saved to the database, then analysis starts automatically.
Step 1 — Context
Build shared context
3 parallel database queries fetch the hand, all actions (with bet-size ratios calculated), and the seat map. Player positions (BTN, BB, etc.) are computed. A hand-evaluator is prepared for strength lookups.
Step 2 — Analysis
9 analyzers run in parallel
Each analyzer gets the shared context and returns a list of tag results. If one analyzer crashes, the others continue — faults are independent.

Step 3 — Validate
Deduplicate results
Hand-level and player-level tags are deduplicated. Sizing tags (one per action) are never deduplicated — the same action can have at most one sizing label anyway.
Step 4 — Store
Atomic replace in database
All previous auto/mistake/sizing tags for this hand are deleted. The new tags are inserted in one operation. Coach tags are never touched.

Tracks how far the hand went. Did everyone fold preflop? Did it reach the river? Was there a showdown?

Classifies the preflop action type. Was it a limped pot? A 3-bet? A squeeze? Includes who opened from button and whether the big blind defended.

Did the preflop raiser bet the flop (c-bet)? Did someone check-raise? Was there a donk bet (non-raiser leading out)? Did someone raise on the river?

Board Analyzer — what did the board look like?
Full characterization of board texture across three dimensions: suit distribution (flush draws), pairing (paired/trips), connectivity (straight draws), and card height.

Mistake Analyzer — potentially questionable plays
Flags common leaks. Tags are attached to the specific player involved, so you can see who made the play, not just that it happened.

Sizing Analyzer — how big were the bets?
Every postflop bet or raise gets a sizing bucket based on its ratio to the pot at the time of the action. One tag per action, attached to both the action and the player.
PROBE_BET <25% THIRD_POT_BET 25–49% HALF_POT_BET 50–79% POT_BET 80–110% OVERBET 110–200% OVERBET_JAM >200%
Hand Strength Analyzer — what did the cards say?
Requires hole cards to be recorded. Evaluates actual hand strength at each street to detect slowplaying, hero calls, missed value, and thin value raises.

04 — What Tags Enable

Downstream Effects

Tags aren't just labels — they power three things in the app today, and have a clear foundation for much more.

Playlist Auto-Creation
Every time a coach applies a manual tag, the system automatically creates (or reuses) a playlist with that tag's name and adds the hand to it. Tag "Bluff" on 5 hands, and a ready-to-drill "Bluff" playlist exists with no extra effort.
Only coach tags trigger this. Auto-tags don't create playlists automatically.
Hand Library Display
In the coach sidebar, every hand in the library shows its tags as colored chips: gold for coach tags, blue/teal for auto/mistake/sizing tags. At a glance, you can see what type of hand it was without replaying it.
Auto-tags and coach tags display with different colors so their source is always visible.
Hand Library Search
The search bar in the hand library scans tag content — searching "bluff" finds hands with the coach tag "Bluff", and searching "CHECK_RAISE" would find hands auto-tagged with that pattern.
Text-based search only. No dedicated tag-filter UI yet.
What tags set up (not yet built)
Player-specific mistake tracking
Because mistake tags are attached to player IDs, it's possible to count how many times "Alice" has been tagged OPEN_LIMP or COLD_CALL_3BET across all hands. This would make for a rich per-player coaching dashboard.
Tag-based filtering in the library
A filter bar — "show me all BLUFF_CATCH hands" or "show only WHALE_POT hands where a CHECK_RAISE occurred" — is fully supported by the database schema, but not yet surfaced as a UI control.
Bet sizing tendencies per player
Sizing tags are tied to specific actions and player IDs. It's possible to compute "Bob bets 60% of pots at POT_BET sizing, almost never uses PROBE_BET" — a full sizing profile per player.
Tag-triggered automatic playlists
Auto-tags could be wired to the same playlist system as coach tags — e.g., every HERO_CALL hand automatically enters a "Hero Calls" replay queue without coach involvement.
05 — Known Limitations

Gaps & Watch-Outs

The tagging system is solid but not complete. These are the places where behavior might surprise you or where features are missing.

Limitation
Manual tags are locked to 6 presets — no custom text
The "TAG HAND" pill shows exactly six fixed options. There is no free-text field. If a coach wants to mark a hand as "5th-street misplay" or "polarized range spot", there's nowhere to type that. The only workaround is to use a playlist with a custom name.
Blind Spot
Hand Strength Analyzer is blind without hole cards
SLOWPLAY, HERO_CALL, VALUE_MISSED, and THIN_VALUE_RAISE are only produced if hole cards were recorded for those players. If hole cards weren't entered into the system, these four tags silently produce nothing — no warning is shown. This affects any hands played without card input.
Limitation
Sizing tags cover postflop only — preflop sizing is untracked
The Sizing Analyzer only runs on postflop actions (flop, turn, river). A preflop open to 10x the big blind gets no sizing tag. There's no way to currently identify players who habitually open too large or min-raise preflop via sizing tags alone (though MIN_RAISE in mistakes partially covers this).
Behavior Note
Re-analyzing a hand replaces all auto-tags, no history kept
Every time the auto-tagging pipeline runs (i.e., every time a hand is reset), it deletes all previous auto, mistake, and sizing tags and inserts a fresh set. There is no tag history or changelog. Coach tags are always preserved, but if the analyzer logic changes in a code update, old hands will get different tags when re-analyzed.
Behavior Note
UNDO_USED doesn't say who undid the action
The UNDO_USED tag is hand-level — it just flags that some action was undone. It doesn't record which player's action was reversed, on which street, or what the action was. You'd need to look at the hand action log manually to find this.
Missing Feature
No filter UI for auto-tags in the hand library
Auto-tags are displayed on each hand but cannot be used to filter the library list. To find "all CHECK_RAISE hands", you'd need to search manually or query the database directly. A tag-filter sidebar or dropdown would unlock this, but doesn't exist yet.
Behavior Note
Auto-tagging only runs on fresh hands — no batch retroactive analysis
Tags are generated immediately after each hand ends. Hands played before the auto-tagging feature was deployed (or before a new analyzer was added) don't get re-analyzed automatically. There is no "re-analyze all past hands" button — this would require a manual script run.
Behavior Note
Coach tag "Mistake" overlaps with the system's mistake tags
There is both a coach-facing "Mistake" preset and a whole category of system-generated mistake tags (OPEN_LIMP, MIN_RAISE, etc.). These are entirely separate — the coach tag is a subjective label, the system tags are rule-based. In the hand library both appear under the same visual style, which could cause confusion about whether a "Mistake" tag was human or system-generated.

Quick Reference

Fast-access facts about how the tagging system works under the hood.

Tag Anatomy
hand_idWhich hand the tag belongs to
tagThe label (e.g. HERO_CALL)
tag_typeOne of: auto, mistake, sizing, coach
player_idIf set: tag applies to this player specifically
action_idIf set: tag applies to a single action (sizing tags)
Tag Scope Levels

Action-levelBoth player ID and action ID attached. "Player X did this on bet #47" (all sizing tags)
