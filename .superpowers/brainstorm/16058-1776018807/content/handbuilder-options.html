A: Toolbar with dropdown

Pros

  • Doesn't push builder content down
  • Clean action hierarchy

Cons

  • Adds a persistent toolbar row

B: Page header with actions

Page-level header above the right panel with title, counts, and action buttons. "+ Add to Playlist" is a primary ghost button. "+ New" is the gold CTA. Scenario cards in sidebar show playlist count indicator (📋 2) so you see at a glance which scenarios are already in playlists.

Pros

  • Cleaner page hierarchy
  • Actions visible even on empty state
  • Playlist count on cards = less hunting

Cons

  • "Add to Playlist" less context-specific
  • Header takes vertical space from builder

Shared improvements (both options)

  • Street-colored left border on scenario cards — instant visual scan of what street each scenario starts on
  • Count badges on Scenarios/Playlists tabs
  • Lucide icons replace emoji (Search, Plus, MoreHorizontal, ListMusic)
  • colors.js tokens everywhere — no hardcoded hex
  • Rounded corners on search input, folder select (border-radius: 6px consistent)
  • Gold focus ring on search input (matches V1 sidebar pattern)
