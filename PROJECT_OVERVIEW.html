Game Workflow

From session start to post-hand review
Normal Hand Flow
1

Players Join

Register/Login → enter table name → join_room assigns stable UUID seat. Coach gets real seat.

2

Coach Configures (Optional)

Specific cards, range strings, board texture constraints, or seat exclusions before dealing.

3

Hand is Dealt

HandGenerator fills gaps (specific cards honored, ranges sampled, rest random). Blinds posted.

4

Street-by-Street Action

60s action timer per player. Coach can Pause, Undo, Force Street, Award Pot at any time.

5

Showdown / Fold

HandEvaluator ranks all live hands. SidePotCalculator distributes chips for all-in scenarios.

6

Persist + Tag

Full hand saved to SQLite. analyzeAndTagHand auto-applies pattern tags. Session stats updated.

Replay Mode Flow
R1

Select Hand

Coach searches Scenario Loader by player/date/tag. Clicks Replay on any result row.

R2

Replay Phase Entered

phase = 'replay'. All hole cards exposed for coaching. Table shows REPLAY badge (blue).

R3

Step Through Actions

◀ Back / Fwd ▶ buttons or scrubber. Active player's seat glows purple. Timer hidden.

R4

Branch to Live (Optional)

Snapshot saved. Phase switches to waiting. BRANCHED badge (amber). Players can now re-play from that exact spot.

R5

Exit / Unbranch

Return to Replay restores snapshot. Exit Replay restores pre-replay stacks and returns to lobby.

Playlist Flow

Coach creates a named playlist of saved hands and activates it. Reset Hand auto-advances to the next scenario in the queue — good for pre-built training sessions.

Full Texas Hold'em Engine

6-max / 9-max, side pots, all streets, proper blind structure.

🎯

Scenario Configuration

Pin specific cards, assign ranges, constrain board texture, exclude players per hand.

Guided Replay Mode

Step through any saved hand, scrub to any action, expose all hole cards for coaching.

Freeze replay mid-hand, play live from that spot, then restore the original line.

Overall Review

Honest top-down assessment
What's Working Well
  • Hand engine is rock-solid — 1,000-hand stress test never fails
  • Side-pot logic handles edge cases (multi-way all-in, dead bets)
  • Replay + Branch mechanic is unique coaching feature — fully implemented
  • Range parser supports full GTO-standard notation
  • Persistent identity (UUID) survives reconnects gracefully
  • Coach has surgical control at every level of the game
  • HTML session reports are self-contained and printable
  • SQLite WAL keeps writes fast even under concurrent reads
  • 800 tests give high confidence in engine correctness
Still Worth Addressing
  • Socket integration tests still thin — socket.integration.test.js file exists but not merged; regressions in index.js are hard to catch automatically
  • Client test coverage is effectively zero for large components (CoachSidebar, PokerTable, StatsPanel)
  • No REST endpoint tests for /api/hands, /api/players, /api/sessions/:id/report
  • Snapshot deep-clone on every action — full JSON round-trip; memory pressure on 30-action hands
  • No starting stack configuration — coach must use adjustStack() per player after joining
  • Dealer rotation doesn't skip disconnected / sitting-out players
Missing / Future Work
  • No mobile-optimised layout for players (designed for desktop)
  • No GTO solver integration or equity calculator
  • No AI opponent or decision feedback overlay
  • No email/push notifications for session invites
  • No multi-table support (one game per server process)
  • No video call integration for remote groups
  • Tournament mode (escalating blinds, eliminations) not yet built
  • Export to PioSolver / GTO+ format for deeper analysis
  • OAuth / SSO — only email+password auth today
Overall Maturity
Production-ready
All tracked bugs resolved. Solid for coached group sessions on LAN or public internet. Main gap: thin socket/client test coverage.
